# Modern Messaging APP (MMAPP)

MMAPP — это современное веб-приложение для обмена мгновенными сообщениями, разработанное на Django. Приложение поддерживает обмен текстовыми сообщениями, изображениями и голосовыми сообщениями в реальном времени.

## Особенности и Функционал

*   **Чаты в реальном времени**: Обмен сообщениями без перезагрузки страницы (используется WebSocket).
*   **Голосовые сообщения**: Запись и отправка голосовых сообщений прямо в браузере.
*   **Статус сообщений**: Индикаторы доставки и прочтения в стиле Telegram/WhatsApp (одна галочка — отправлено, две — прочитано).
*   **Мультимедиа**: Отправка и встроенный просмотр изображений.
*   **Статус онлайн**: Отображение статуса "в сети" и времени последнего посещения для пользователей.
*   **Управление профилем**:
    *   Редактирование профиля (имя пользователя, bio).
    *   Загрузка аватара пользователя.
    *   Смена пароля.
*   **Приватные чаты**: Создание и ведение личных переписок.

## Технологический стек

*   **Backend**: Python, Django, Django Channels (WebSockets).
*   **Frontend**: HTML, JavaScript, Tailwind CSS (для стилизации), Alpine.js.
*   **База данных**: SQLite (по умолчанию), совместимость с PostgreSQL.

## Инструкция по установке и запуску

### 1. Предварительные требования
Убедитесь, что у вас установлен **Python** (версии 3.8 или выше).

### 2. Клонирование и настройка окружения

Рекомендуется использовать виртуальное окружение:

```bash
# Создание виртуального окружения
python -m venv venv

# Активация виртуального окружения
# Windows:
.\venv\Scripts\activate
# Linux/macOS:
source venv/bin/activate
```

### 3. Установка зависимостей

Установите необходимые библиотеки из файла `requirements.txt`:

```bash
pip install -r requirements.txt
```

### 4. Настройка базы данных

Примените миграции для создания структуры базы данных:

```bash
python manage.py migrate
```

### 5. Запуск сервера

Запустите сервер разработки Django:

```bash
python manage.py runserver
```

После запуска приложение будет доступно по адресу: [http://127.0.0.1:8000/](http://127.0.0.1:8000/)

---

**Примечание**: Для работы WebSocket в режиме разработки используется `InMemoryChannelLayer`. Для продакшн-окружения рекомендуется переключиться на Redis.
